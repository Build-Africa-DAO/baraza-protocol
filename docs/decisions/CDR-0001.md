# CDR-0001 — Legacy single-chain Stellar architecture

- **Type:** Chain Decision Record
- **Status:** SUPERSEDED by the current architecture in `AGENTS.md`
- **Original date:** 2026-06-24
- **Restored:** 2026-07-03 after the public-history reset

## Historical decision

The v1 protocol was launched as a Stellar-only Soroban suite. Its membership,
governance, and vesting contracts were deployed to Stellar testnet, with the
vesting contract bound to a TBRZA test asset. EVM and Solana work was deferred
at that time.

## Current status

This decision no longer describes the repository's architecture. Stellar is the
primary settlement layer, Solana is the governance layer, and EVM is a
secondary read-only governance-token layer. The current Soroban v2 workspace
also differs from the deployed v1 interfaces.

The v1 testnet IDs and deployer are preserved in
`contracts/stellar/addresses/testnet-v1.json`. The unversioned
`addresses/<network>.json` format belongs to the current five-contract v2 deploy
script and must not be populated with v1 IDs.

## Consequences

- The v1 deployment remains valid evidence of the June 2026 testnet launch.
- The v1 deployment must not be presented as a deployment of the current v2
  source tree.
- A new v2 deployment must generate its own `addresses/testnet.json` using
  `contracts/stellar/scripts/deploy.sh`.
